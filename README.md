## Описание проекта
Проект представляет собой программно-аппаратный комплекс для симуляции и интеллектуального планирования производственных процессов (APS — Advanced Planning and Scheduling). Он состоит из трех основных частей:
1. **Java Spring Boot сервер** — ядро системы. Содержиет логику APS-алгоритма, OPC UA сервер для обмена данными с оборудованием и REST API для веб-интерфейсов.
2. **React интерфейс (Веб-дашборд)** — интерактивный цифровой двойник, отображающий текущий шаг алгоритма планирования, очередь задач MES и детальную статистику по оборудованию в реальном времени.
3. **PyGame симулятор (Python)** — визуальный симулятор гальванической линии, выступающий в роли исполнительного оборудования.

## Реализация APS-алгоритма
Алгоритм планирования полностью соответствует разработанной блок-схеме и реализует динамическую приоритизацию партий. Весь процесс разбит на фазы:
1. **Проверка активности** (`CHECK_ACTIVITY`) — проверка состояния оборудования (через OPC UA).
2. **Очередь** (`CHECK_QUEUE`) — проверка наличия задач в расписании (`schedule.json`).
3. **Приоритет** (`CALC_PRIORITY`) — математическое вычисление приоритета каждой партии на основе её веса, материала и срочности.
4. **Загруженность** (`CALC_UTILIZATION`) — оценка текущей занятости ванн и исполнительного оборудования.
5. **Время** (`CALC_PROCESSING_TIME`) — расчет времени обработки с учетом коэффициентов материала.
6. **Интервалы и Сопоставление** (`CALC_FREE_INTERVALS`, `COMPARE_SELECTION`) — поиск свободных окон для включения партии без простоя.
7. **Утверждение** (`APPROVE_SEQUENCE`) — принятие решения о запуске или отказе (с возможным переходом в ручной режим).
8. **Исполнение** (`EXECUTING`) — эмуляция времени выполнения заказа с автоматическим переводом задачи в статус "В работе".

## Вычисление метрик
- **Приоритет партии:** Рассчитывается динамически. Зависит от материала (Сталь, Медь, Алюминий имеют разные весовые коэффициенты) и массы деталей. В случае ручного вклинивания оператором, приоритет принудительно устанавливается на 999.
- **Загруженность оборудования:** Каждое устройство (ванна, кран) имеет свою "емкость" (Capacity). Загруженность вычисляется как отношение текущей нагрузки к общей емкости. При назначении операций загруженность растет.
- **Время обработки:** Базовое время работы устройства умножается на коэффициент материала (например, Медь обрабатывается дольше - коэффициент 1.3, Алюминий быстрее - 0.8) и коэффициент массы детали.
- **Отказы и коллизии:** В алгоритм заложена вероятность конфликта приоритетов для симуляции реальных производственных коллизий, которые требуют пересчета. Если лимит пересчетов превышен, система переходит в ручной режим управления оператором.

## Структура файла schedule.json
Файл `schedule.json` является локальной базой данных, содержащей объекты партий деталей. Структура состоит из двух массивов:
- `queue`: очередь деталей, ожидающих обработки.
- `current_tasks`: партии, которые были одобрены алгоритмом APS и взяты в работу исполнительным оборудованием.
Пример структуры одной партии:
```json
{
  "id": "BATCH_1784467509118_580",        // Уникальный идентификатор партии
  "name": "Авто_Сталь_376",               // Название партии (генерируется или вводится оператором)
  "material": "Сталь",                    // Материал деталей (Сталь, Медь, Алюминий)
  "weight": 57.99,                        // Общая масса партии в кг
  "priority": 22,                         // Итоговый вычисленный приоритет
  "urgencyScore": 1,                      // Оценка срочности (чем больше, тем важнее)
  "materialScore": 10,                    // Оценка на основе типа материала
  "weightScore": 11,                      // Оценка на основе массы партии
  "assignedLineId": "LINE_GALVANIC_1",    // Назначенная производственная линия
  "status": "SCHEDULED",                  // Статус (SCHEDULED, IN_PROGRESS, COMPLETED)
  "totalProcessingTimeSec": 5002,         // Суммарное время обработки в секундах
  "steps": [                              // Технологическая карта (список шагов)
    {
      "equipmentId": "BATH_DEGREASE",     // Идентификатор оборудования (ванны)
      "operation": "01 Обезжиривание",    // Название операции
      "durationSec": 619,                 // Вычисленное время операции
      "completed": false                  // Флаг завершения операции
    }
  ],
  "createdAt": 1784467509155,             // Время создания (timestamp)
  "scheduledStartTime": 1784467529220,    // Запланированное время старта
  "scheduledEndTime": 1784472531220       // Запланированное время окончания
}
```

## Запуск проекта
1. Запуск сервера: `mvn spring-boot:run` в папке `java-server`
2. Запуск интерфейса: `npm run dev` в корне проекта (порт 3000)
3. Запуск симулятора: `python main.py` в папке `SimOPC`
